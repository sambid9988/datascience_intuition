Theres a really great paper by Fayyad and Irani on how to do this (Multi-Interval Discretization of Continued Valued Attributes PDF available here).

First of all, there is a simple algorithm that works but is slow: consider every observed value of the continuous variable. So if I see 1.4,1.7,4.8 in the data across all classes, I try each value as a potential cutpoint.

What they prove there is that you dont need to look at all possible values, just sort your values and then only consider the cutpoints where the class changes. So if 1.4 is class A and 1.7 is class A and 4.8 is class B, then you would consider only a cut in 1.7 to 4.8 range and any cut in that range is guaranteed to give you the same training set error. Theres no point trying a cut between 1.4 and 1.7 because its guaranteed to have more entropy than the cut between 1.7 and 4.8.

Different algorithms do different things; some choose the midpoint between 1.7 and4.8, some just say < 4.8.

It depends on the algorithm, of course.

Ive seen algorithms where they have candidate cutoff values equal to the midpoints of all intervals between values of that variable in the data. For example, if the data consists of the values [2, 4, 9, 10], the candidate values would be 3, 6.5 and 9.5. For each of the candidates a statistical test is performed to determine the improvement of the quality of the model, and if a split has a significant improvement, it is introduced.
